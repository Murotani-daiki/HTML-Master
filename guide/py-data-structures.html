データ構造

情報をひとまとめにして管理し、効率よくプログラムを動かそう

情報をどうまとめるか？

プログラムでは単一の数値や文字列だけでなく、「複数のデータの集まり」を扱うことがよくあります。Pythonには、用途に合わせて使い分けられる強力なデータ構造が備わっています。

リスト [list]

順番があるデータの集まり。インデックス(0から始まる番号)で管理します。中身を自由に変更できます。

辞書 {dict}

「キー」と「値」をセットにして保存。名簿のように、特定の名前から情報を素早く引き出すのに最適です。

1. リストの基本

fruits = ["Apple", "Banana"]
fruits.append("Cherry") # 追加
print(fruits[0])        # "Apple"を取得

2. 辞書の基本

user = {"name": "Taro", "age": 20}
print(user["name"]) # "Taro"を取得

3. タプル (tuple)

丸カッコ `( )` を使います。リストと似ていますが、**一度作った後で中身を変更できない**という特徴があります。設定値など、変えたくないデータを扱うのに適しています。

まとめ

それぞれのデータ構造には得意・不得意があります。データの順序が大事ならリスト、名前で管理したいなら辞書、不変性を保ちたいならタプル、というように使い分けていきましょう。
